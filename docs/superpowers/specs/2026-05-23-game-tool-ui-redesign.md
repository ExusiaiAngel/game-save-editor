# GameSaveEditor UI 重设计规范

日期: 2026-05-23 | 版本: v1 | 状态: 已确认

---

## 一、概述

### 1.1 设计目标

对 GameSaveEditor 的 GUI 进行**布局重构 + 视觉美化**改造，从当前的"固定双面板调试工具"风格升级为"模块化侧栏导航 + 游戏主题暗色 UI"的专业工具风格。

### 1.2 范围与约束

| 维度 | 约束 |
|------|------|
| **框架** | 保持 Rust + egui 0.31 + eframe 0.31，不切换前端技术栈 |
| **引擎适配** | 保持现有所有引擎支持（RPG Maker MV/MZ/NW.js, Ren'Py, Unreal, Unity, Godot） |
| **功能覆盖** | 全部 6 个模块视图；现有功能不丢失 |
| **平台** | Windows 桌面为主，兼容 macOS/Linux |
| **与 V2 spec 关系** | 互补关系 — 本 spec 定义布局/视觉，V2 spec 定义功能改进项。合并后统一执行 |

### 1.3 当前 UI 痛点

1. 固定双面板布局 — 字段列表纵向空间受限，信息架构僵化
2. 视觉风格粗糙 — egui 默认浅色主题，无暗色模式，像"调试工具"而非"产品"
3. 面板功能不对等 — 存档面板有搜索/分类，实时面板缺失
4. 无全局概览 — 缺少仪表盘首页，用户打开后无引导
5. 备份管理缺失 — 备份文件散落在文件系统，无集中管理视图

### 2.2 ViewState 枚举

```rust
#[derive(Clone, PartialEq)]
pub enum AppView {
    Dashboard,       // 仪表盘（默认首页）
    Toolbox,         // 工具箱
    Settings,        // 设置
    SaveEditor,      // 存档编辑
    RealtimeEditor,  // 实时修改
    BackupManager,   // 备份管理
}
```

### 2.3 导航规则

- 无游戏加载时，仅仪表盘、设置可访问
- 选中游戏后，存档编辑 / 实时修改 / 备份管理解锁
- 打开存档文件后，才能进行字段编辑
- 实时修改需要成功连接才能写入

### 2.4 AppState 新增字段

```rust
pub struct AppState {
    // ...现有字段...
    pub active_view: AppView,              // 当前选中视图
    pub sidebar_collapsed: bool,           // 侧栏折叠状态

    pub dark_mode: bool,                   // [F25] 暗色主题（默认 true）
    pub recent_games: Vec<String>,         // [F17] 最近游戏列表
    pub backups: Vec<BackupInfo>,          // [F16] 备份列表
}
```

## 四、六视图设计

### 4.1 ① 仪表盘 (Dashboard)

**位置**：默认首页，无游戏时亦可显示

**组件**：

- **快速操作区**（水平按钮组）
  - `[打开游戏...]` — 调用 rfd 文件对话框选择游戏目录
  - `[打开存档...]` — 直接打开存档文件
  - `[最近游戏 ▾]` — 下拉最近 5 个游戏路径

- **游戏信息卡片**
  - 游戏标题、引擎类型（图标+名称）
  - 存档统计（存档数量、备份数量、总字段数）
  - 连接状态指示器（●/◌/○）+ 端口号
  - `[连接]` / `[断开]` 按钮
  - `[注入插件]` 按钮（支持的引擎）

- **最近存档列表**（表格）
  - 文件名 | 大小 | 修改时间 | 操作([加载])
  - 自动发现 game_dir 下的存档文件
  - 点击加载跳转到存档编辑视图

**状态**：
- 无游戏：显示空状态引导 `"选择一个游戏目录开始"` + `[打开游戏...]` 大按钮
- 无存档：游戏信息显示，存档列表为空

**位置**：从仪表盘或侧栏导航进入

**布局（自上而下）**：
```
┌─ 连接管理栏 ──────────────────────────────┐
│ ● 已连接  127.0.0.1:19999  [断开]  [注入插件]  │

│   金币     金币     [9999]   🔓     9999   │
│   开关     大门     [✓]     🔒     ✓      │

**新增功能**（来自 V2 spec F4-F13）：
- 搜索 + 分类过滤（F6-F7）

- 移除 20 字段限制，全量 + ScrollArea（F9）

- 编辑值后自动发送 WriteField 命令

- 连接状态变化时自动禁用/启用编辑控件

### 4.4 ④ 备份管理 (Backup Manager)

**位置**：侧栏导航

**布局**：
```
┌─ 源文件选择 ──────────────────────────────┐
│ 存档: [file1.rpgsave ▾]                    │
└──────────────────────────────────────────┘

┌─ 备份列表 ───────────────────────────────┐
│   时间戳              大小     操作         │
│   ──────────────────────────────────     │
│   2026-05-22 14:30:15  128KB  [恢复] [删除]│
│   2026-05-21 09:15:42  126KB  [恢复] [删除]│
│   2026-05-20 18:45:03  124KB  [恢复] [删除]│

┌─ 备份对比 (选中两个备份时显示) ─────────────┐

**功能**：
- 发现存档文件的所有 `.bak` / `.orig` 备份

- 选中两个备份时显示差异对比

### 4.5 ⑤ 工具箱 (Toolbox)

**位置**：侧栏导航，无需加载游戏即可使用

**子工具**（纵向排列，每个工具一个可折叠 Section）：

1. **LZString 压缩/解压**
   - 两个输入框：原始 JSON 输入 / 压缩文本输入
   - `[压缩]` → 输出 Base64 压缩结果，[复制] 按钮
   - `[解压]` → 输出 JSON 结果，[复制] 按钮
   - 支持 LZString Base64 格式（RPG Maker MV 兼容）

2. **Base64 编解码**
   - 编码：文本输入 → Base64 输出
   - 解码：Base64 输入 → 文本输出
   - 各带 [复制] 按钮

3. **存档完整性检查**
   - 文件选择器 `[浏览...]` + 文件路径显示
   - `[检测]` 按钮
   - 检查清单：JSON 合法 / _format 字段匹配 / magic bytes 一致 / 必要字段存在
   - 结果以 ✓/✗ 列表展示，附带详细问题描述

4. **游戏目录扫描器（手动）**
   - 目录输入框 + `[浏览...]` + `[扫描]`
   - 显示结果：引擎类型、存档目录路径、开关数、变量数、物品数
   - `[导出 JSON]` 按钮

### 4.6 ⑥ 设置 (Settings)

**位置**：侧栏导航，始终可访问

**分区**：

1. **外观** — 🌙/☀ 主题切换（即时生效）
2. **连接** — 默认端口（可编辑数字输入）、主机地址
3. **最近游戏** — 列表展示 + `[清除]` 按钮 + 每个条目 `[移除]`
4. **关于** — 应用名称、版本号、引擎支持列表、GitHub 链接

## 五、底部快捷操作栏 (QuickBar)

### 5.1 设计原则

- 固定在内容区底部，不受字段表格滚动影响
- 内容根据当前 `active_view` 动态切换
- 仅存档编辑和实时修改视图显示完整快捷栏

### 5.2 存档编辑视图快捷栏

```
┌──────────────────────────────────────────────────────────────┐
│ 💾 [保存 (3处)]  ↩ [撤销全部]   📤 [复制选中 → 实时]          │
│ 🔍 搜索: [...]  ✕  分类: [全部 ▾]  跳转 ID: [___]           │
│ 共 1234 字段  |  3 处修改未保存  |  D:\save.rpgsave          │
└──────────────────────────────────────────────────────────────┘
```

### 5.3 实时修改视图快捷栏

```
┌──────────────────────────────────────────────────────────────┐
│ ● 已连接 :19999  [断开]  [注入插件]                           │
│ ▶ 自动刷新  📥 [手动刷新]  🔒 3 个字段已锁定                   │
│ 📥 [复制选中 ← 存档]   📤 [复制选中 → 存档]                    │
│ 🔍 搜索: [...]  ✕  分类: [全部 ▾]  跳转 ID: [___]           │
│ 共 500 字段  |  已连接: RPG Maker MV                         │
└──────────────────────────────────────────────────────────────┘
```

### 5.4 跨面板拷贝逻辑

| 方向 | 触发 | 行为 |
|------|------|------|
| 存档 → 实时 | `📤 [复制选中 → 实时]` | 发送 `WriteField(field_id, save_value)` 到 TCP bridge |
| 实时 → 存档 | `📥 [复制选中 ← 存档]` | 更新 `save_data[field_id] = live_value`，标记 dirty |
| 批量 | 支持 Ctrl/Shift 多选行，批量拷贝 |

### 5.5 其他视图

| 视图 | 快捷栏内容 |
|------|-----------|
| 仪表盘 | 仅状态信息行 "未加载游戏" / "已加载: RPG Maker MV | 未连接" |
| 工具箱 | 状态信息行 "工具箱 — 独立工具，无需加载游戏" |
| 备份管理 | 状态信息行 "备份管理 — 共 N 个备份文件" |
| 设置 | 状态信息行 "设置" |

### 6.5 暗色/亮色切换

- 侧栏底部或设置页面提供 🌙/☀ 切换按钮
- 切换即时生效（`ctx.set_visuals(Visuals::dark())` / `ctx.set_visuals(Visuals::light())`）
- 配色使用 `egui::Visuals` + 自定义颜色覆盖
- 默认暗色模式

### 6.6 主题配置结构

```rust
pub struct ThemeConfig {
    pub dark_mode: bool,
}

impl ThemeConfig {
    pub fn apply(&self, ctx: &egui::Context) {
        let mut visuals = if self.dark_mode {
            egui::Visuals::dark()
        } else {
            egui::Visuals::light()
        };
        // 覆盖关键颜色
        visuals.widgets.noninteractive.bg_fill = if self.dark_mode {
            Color32::from_rgb(22, 27, 34)  // #161b22
        } else {
            Color32::from_rgb(246, 248, 250)  // #f6f8fa
        };
        // ... 更多覆盖 ...
        ctx.set_visuals(visuals);
    }
}
```
